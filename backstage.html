Building screens

Build a dashboard with live data

Use when you need a summary screen with stat cards and program overviews.

Dashboard
Build a dashboard screen using our design system. Include: - Four stat cards from db.dashboard.summary() - A program overview section from db.programs.list() - Use shadcn/ui Card, Badge components - Colors only from Radix CSS variables — no hex values - Use Figtree font, Tailwind spacing

Build a client list view

Use when you need a filterable list with risk indicators.

Client List
Build a client list view that: - Loads clients from db.clients.list() - Shows name, program, risk level Badge, assigned clinician - shadcn/ui Input for search, filters client-side - High risk clients get a red Badge variant="destructive" - shadcn/ui Card for each row or as the container

Build a client detail view

Use when you need a single client's full clinical record.

Client Detail
Build a client detail view for 'CL-005' that: - Loads via db.clients.get('CL-005') - Shows name, pronouns, DOB, phone, clinician, insurance in a shadcn/ui Card - Tabs (shadcn/ui Tabs) for: Diagnoses, Medications, Allergies, Care Journey - Each tab loads from db.diagnoses / db.medications / db.allergies / db.journey - Use Radix color variables for risk-level badges

Build a referral management screen

Use when prototyping the closed-loop referral workflow.

Referrals
Build a referral management screen: - Loads db.referrals.list(), shows pending/in-progress/completed counts in Cards - Table rows (shadcn/ui Table) with client name, from→to path, urgency Badge, status - shadcn/ui Dialog for "+ Create Referral" form - Form: client select, from/to program, urgency, reason, notes - On submit: db.referrals.create({...})

Build a warm line call log

Use when prototyping crisis line or call management features.

Warm Line
Build a warm line call log: - Loads db.calls.list(), shows each call in a Card row - Priority Badge: routine=default, urgent=warning, crisis=destructive - Crisis calls: red Alert (shadcn/ui Alert) shown below the row - shadcn/ui Dialog for "+ Log New Call" - Form: caller name, phone, relationship, priority, summary, outcome, follow-up Switch - On submit: db.calls.create({...})
Common UI patterns

Add a sidebar navigation

Consistent nav shell for any prototype screen.

Navigation
Add a left sidebar navigation: - "One Chorus" app name + "Mental Health Program Operations" subtitle - Nav items: Dashboard, Warm Line, Clients, Referrals — each with fa-regular icon - Active item highlighted with var(--blue-9) accent - Top bar: shadcn/ui Input for search, program name, settings icon, today's date - Sidebar background: var(--mauve-2), borders: var(--mauve-6), width: 180px

Add an empty state

For lists or sections that might return no data.

Empty State
Add an empty state when there is no data: - fa-regular icon centered, contextually appropriate - Short heading, one-line supporting description - Optional shadcn/ui Button (variant="default") for a primary action - Icon color: var(--mauve-9), text: var(--mauve-11)

Fix spacing or layout

When something feels too tight or too loose.

Iteration
The [section name] feels too cramped / spread out. Adjust spacing using Tailwind spacing utilities (p-2 through p-16, gap-2 through gap-8) — no arbitrary pixel values. Keep all other styles unchanged.
