Building screens
Dashboard

Build a dashboard with live data

Use when you need a summary/overview screen with stat cards and lists.

Build a dashboard screen using our theme.css design tokens. Include: - Four stat cards across the top pulling from db.dashboard.summary() - A list section pulling real data from Supabase using db.js - Use .card, .badge, and .btn classes from our design system - No arbitrary colors or font sizes — only use CSS variables from theme.css
Client List

Build a client list view

Use when you need to show a list of clients with filters or search.

Build a client list view that: - Loads clients from Supabase using db.clients.list() - Shows each client's name, program, risk level badge, and assigned clinician - Includes a search input that filters the list client-side - High risk clients should show a red badge using .badge-red - Use our theme.css card and typography classes throughout - Keep it simple — this is a prototype, not production code
Client Detail

Build a client detail / profile view

Use when you need to show a single client's full record.

Build a client detail view for client ID 'CL-005' that: - Loads the client using db.clients.get('CL-005') - Shows name, pronouns, DOB, phone, assigned clinician, insurance - Loads and displays their diagnoses using db.diagnoses.forClient() - Loads and displays their medications using db.medications.forClient() - Loads and displays their allergies using db.allergies.forClient() - Shows a care journey timeline using db.journey.forClient() - Use our card, badge, and typography classes from theme.css
Referrals

Build a referral management screen

Use when prototyping the referral workflow.

Build a referral management screen that: - Loads all referrals using db.referrals.list() - Shows stat cards for pending, in-progress, completed, declined counts - Lists referrals with client name, from/to program path, urgency badge, and status - Includes a "+ Create Referral" button that opens a modal form - The form should have: client select, from program, to program, urgency level, reason, notes - On submit, call db.referrals.create() with the form data - Use our theme.css design tokens throughout
Warm Line

Build a warm line call log screen

Use when prototyping crisis line or call management features.

Build a warm line call log screen that: - Loads calls using db.calls.list() - Shows each call with caller name, priority badge (routine/urgent/crisis), phone, summary, outcome, handled by, duration - Crisis priority calls should have a red warning banner below them - Includes a "+ Log New Call" button that opens a modal form - The form needs: caller name, phone, relationship (self/family/other), priority level, call summary, outcome, follow-up required toggle - On submit, call db.calls.create() with the form data - Use our theme.css design tokens throughout
Common UI patterns
Navigation

Add a sidebar navigation

Use to add consistent nav to any prototype screen.

Add a left sidebar navigation to this prototype with: - App name "One Chorus" and subtitle "Mental Health Program Operations" at the top - Nav items: Dashboard, Warm Line, Clients, Referrals — each with a Font Awesome icon - One item should be marked active (highlighted with our primary blue) - A top bar with a global search input, current program name, settings icon, and today's date - Use our mauve color scale for the sidebar background and borders - Keep the sidebar fixed at 180px wide
Empty State

Add an empty state

Use when a list or section might have no data.

Add an empty state to this section that shows when there is no data. It should have: - A Font Awesome icon centered (fa-regular, pick something contextually appropriate) - A short heading like "No clients yet" - A one-line description - Optionally a primary button to take action - Use our mauve-9 color for the icon and mauve-11 for the text
